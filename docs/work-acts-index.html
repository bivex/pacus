Этот файл — точка входа в пакет документов по архитектуре, схеме БД, ERD, API, миграциям и хранению печатных HTML-артефактов.

1. Порядок чтения

  1. Сначала общая архитектура и доменная модель.
  2. Потом физическая схема БД и ERD.
  3. Затем API/операции и рендеринг артефактов.
  4. После этого — план миграций и внедрения.

3. Ключевые решения

  • Акт — это доменная сущность, а не просто HTML-файл.
  • Каждая версия акта фиксируется отдельно в work_act_revision.
  • HTML печатной формы хранится как неизменяемый артефакт версии.
  • После подписи версия и связанные артефакты не изменяются.
  • Исправления после подписи оформляются новым актом, связанным с исходным.
  • Для быстрого старта допустим SQLite, но immutable-историю и статусы всё равно должен контролировать backend-слой.

4. Следующий технический шаг

После утверждения этого пакета можно использовать db/sqlite/work_acts_schema.sql, подключить SQLite к NocoDB и затем готовить backend-contract для операций: фиксация ревизии, генерация HTML, отправка, подписание и корректировка.

Для быстрого demo-старта также подготовлен db/sqlite/work_acts_seed_demo.sql с данными по tenant, контрагентам, договорам, актам, версиям и артефактам.
