Спецификация: Домен «Проекты»

Документ описывает доменную модель проектов, FSM статусов, инварианты БД, обязанности приложения и HTML-артефакты для системы Pacus.

1. Назначение домена

  • Проект группирует акты работ одного исполнителя с одним контрагентом.
  • Акт может быть привязан к проекту опционально (project_id nullable).
  • Проект не привязан к конкретному договору — один проект может охватывать несколько договоров.
  • Все акты внутри проекта принадлежат одному tenant и одному counterparty.

5. Обязанности приложения (что БД не проверяет)

  • Запрет привязывать новые акты к completed/cancelled проекту.
  • Обязательность finished_on при переходе в completed.
  • При смене counterparty_id или tenant_id в акте с project_id — проверить совместимость (триггеры срабатывают только на UPDATE OF project_id).
Известная дыра в схеме: триггеры trg_work_act_project_*_match срабатывают только при UPDATE OF project_id. Если изменить work_act.counterparty_id при уже установленном project_id — БД это пропустит. Приложение обязано запрещать такое изменение.

6. Правило транзакции

Каждое изменение статуса выполняется в одной транзакции, содержащей UPDATE и INSERT в историю.

BEGIN;
UPDATE project
SET status = :new_status, finished_on = :finished_on, updated_at = :now_utc
WHERE id = :project_id;

INSERT INTO project_status_history (id, project_id, from_status, to_status, changed_by, changed_at, reason)
VALUES (:id, :project_id, :old_status, :new_status, :user, :now_utc, :reason);
COMMIT;

from_status — читать до UPDATE в той же транзакции.

АртефактПутьСодержание

Запуск генератора

python3 scripts/gen_artifacts.py

Опциональные аргументы: --db path/to/db.sqlite, --out path/to/artifacts/

Все генерируемые файлы: самодостаточные (inline CSS), относительные ссылки между артефактами, валюта задаётся через const CURRENCY = '$' в каждом файле.
