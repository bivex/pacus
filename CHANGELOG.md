## [0.3.0] - 2026-03-21

### Added
- **Projects domain** — `project` table with status FSM (`active → on_hold → completed / cancelled`), `project_status_history` (append-only).
- `work_act.project_id` — optional FK linking acts to a project; tenant and counterparty consistency enforced by triggers.
- **6 new triggers** on project and work_act: status FSM guard, append-only history, cross-entity tenant/counterparty match.
- **2 new views**: `v_nocodb_project_overview` (aggregates), `v_nocodb_project_work_acts` (act chronology with print_url).
- `db/sqlite/projects_schema.sql` — migration applied on top of work_acts schema.
- `db/sqlite/projects_seed_demo.sql` — 2 demo projects with status history, acts assigned.
- **HTML artifact generator** — `scripts/gen_artifacts.py`: queries SQLite, generates all project cards, act audit trails, and `index.html` with correct relative links. Self-contained output (inline CSS + JS currency var).
- **Project card** HTML artifact per project: status history table + acts chronology with amounts.
- **Act audit trail** HTML artifact per act: status timeline, revision list, related documents.
- Currency variable (`const CURRENCY = '$'`) in every generated HTML — change one line to switch all symbols.
- **`.agents/` skill system** — `SKILLS.md` index + `skills/projects.md`, `skills/work_acts.md`, `skills/db_invariants.md` for AI-agent guidance.
- **Schema tests** — `tests/test_db_schema.py`: 88 tests covering every constraint and trigger.
- **Mutation tests** — `tests/test_mutations.py`: 22 tests proving each guard is load-bearing (removing it breaks the invariant).

### Changed
- `data/artifacts/index.html` is now generated by `gen_artifacts.py` (relative links, inline CSS, no external dependencies).

### Fixed
- All artifact HTML files now embed CSS inline — work when opened directly from filesystem without a running HTTP server.

## [0.2.0] - 2026-03-08

### Added
- **Integration inbox** — `integration_inbox_work_act` table with idempotency key, FSM (`pending → processing → imported / error`), trigger `trg_integration_inbox_no_imported_without_act`.
- `scripts/import_inbox_work_acts.py` — inbox processor: validates, creates draft acts, updates import status.
- `scripts/run_inbox_import.sh` — runner for the inbox processor.
- NocoDB views for inbox: `Inbox All`, `Inbox`, `Imported`, `Errors`, `Ready to Freeze`.

## [0.1.0] - 2026-03-08

### Added
- **Core work acts schema** (`db/sqlite/work_acts_schema.sql`):
  - Tables: `tenant`, `counterparty`, `contract`, `work_act`, `work_act_item`, `work_act_revision`, `work_act_status_history`, `document_artifact`, `audit_event`.
  - Status FSM (`draft → generated → sent → signed → corrected / cancelled`) enforced by trigger.
  - Immutable revision pattern: `is_immutable = 1` blocks UPDATE/DELETE on revision and its artifacts.
  - Append-only guards on `work_act_status_history` and `audit_event`.
  - Signed revision immutability: `signed_revision_id` cannot be changed once set.
  - Partial unique index `uq_work_act_revision_current` — only one `is_current = 1` revision per act.
  - NocoDB-oriented views: `v_nocodb_work_act_overview`, `v_nocodb_work_act_drafts`, `v_nocodb_print_links`, `v_nocodb_work_act_corrections`, `v_nocodb_work_act_revisions_readonly`, `v_nocodb_work_act_status_history_readonly`, `v_nocodb_document_artifacts_readonly`.
- Demo seed data (`db/sqlite/work_acts_seed_demo.sql`): 5 acts across all statuses with revisions, artifacts, status history, audit events.
- Print artifacts (`data/artifacts/`): self-contained HTML act forms, shared `print.css`, print center `index.html`.
- **One-click bootstrap**: `scripts/start_work_acts_demo.sh` — rebuilds DB, starts print server (port 4000), starts NocoDB (port 1233), bootstraps auth/base/source, runs Meta Sync.
- `scripts/start_print_artifacts_server.sh` — standalone print server.
- `scripts/stop_work_acts_demo.sh` — teardown.
